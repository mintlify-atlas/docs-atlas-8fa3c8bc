Warden supports three scheduling patterns: **cron expressions** (recurring), **one-shot timestamps** (at), and **interval-based** (every). All jobs are stored in Supabase and polled every 30 seconds.

## Scheduling Architecture

From `~/workspace/source/src/cron.ts:10-72`:

```typescript
const POLL_INTERVAL_MS = 30_000; // 30 seconds

async function fireDueJobs(): Promise<void> {
  const dueJobs = await pollDueCronJobs();
  for (const job of dueJobs) {
    // Create a task from the cron job
    const task = await insertTask({
      instruction: job.instruction,
      metadata: { ...(job.task_metadata ?? {}), cron: true },
    });
    
    // Compute next run time
    const nextRun = computeNextRun(job);
    await markCronJobRun(job.id, task.id, nextRun);
    
    // Handle one-shot jobs
    if (job.schedule_type === "at" || !nextRun) {
      if (job.delete_after_run) {
        await deleteCronJob(job.id);
      } else {
        await updateCronJob(job.id, { enabled: false });
      }
    }
  }
}
```

<Note>
  All cron-generated tasks include `{"cron": true}` in their metadata for filtering and auditing.
</Note>

### Cron Expression Format

```
┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of week (0 - 6) (Sunday to Saturday)
│ │ │ │ │
* * * * *
```

### Timezone Support

From `~/workspace/source/src/cron-utils.ts:8-18`:

```typescript
export function computeNextRun(job: CronJob, from?: Date): string | null {
  const now = from ?? new Date();
  
  switch (job.schedule_type) {
    case "cron": {
      const interval = CronExpressionParser.parse(job.cron_expression, {
        currentDate: now,
        tz: job.cron_timezone || "UTC",  // Defaults to UTC
      });
      return interval.next().toISOString();
    }
  }
}
```

<Warning>
  Always specify `--tz` for time-sensitive jobs. IANA timezone format (e.g., "America/Los_Angeles", "Europe/London", "Asia/Tokyo").
</Warning>

## Schedule Type: At (One-Shot)

Fire once at a specific datetime, then auto-disable or delete.

### Basic Usage

```bash
# Deploy at specific time
npx tsx src/cron-cli.ts add \
  --name "weekend-deploy" \
  --at "2026-03-15T02:00:00Z" \
  --instruction "Deploy latest build to production"

# Reminder with custom timezone
npx tsx src/cron-cli.ts add \
  --name "meeting-reminder" \
  --at "2026-03-10T14:00:00" \
  --tz "America/New_York" \
  --instruction "Send reminder: All-hands meeting in 15 minutes"
```

### Auto-Cleanup Behavior

From `~/workspace/source/src/cron.ts:36-43`:

```typescript
if (job.schedule_type === "at" || !nextRun) {
  if (job.delete_after_run) {
    await deleteCronJob(job.id);
    console.log(`[cron] Deleted one-shot job "${job.name}" (${job.id})`);
  } else {
    await updateCronJob(job.id, { enabled: false });
    console.log(`[cron] Disabled one-shot job "${job.name}" (${job.id})`);

}
```

<Steps>
  <Step title="Job Fires">
    Task is created and queued when timestamp is reached
  </Step>
  <Step title="Auto-Cleanup">
    By default (`--delete-after-run` is true for `--at` jobs), the cron job is **deleted** from the database
  </Step>
  <Step title="Keep History">
    Omit `--delete-after-run` to disable instead of delete (useful for auditing)
  </Step>
</Steps>

## Schedule Type: Every (Interval)

Repeat at fixed intervals using human-friendly syntax.

### Interval Syntax

From `~/workspace/source/src/cron-utils.ts:36-52`:

```typescript
const INTERVAL_UNITS: Record<string, number> = {
  s: 1000,           // seconds
  m: 60 * 1000,      // minutes
  h: 60 * 60 * 1000, // hours
  d: 24 * 60 * 60 * 1000, // days
};

export function parseInterval(str: string): number {
  const match = str.match(/^(\d+(?:\.\d+)?)\s*([smhd])$/i);
  if (!match) throw new Error(`Invalid interval "${str}". Use format like 30s, 5m, 2h, 1d`);
  const value = parseFloat(match[1]);
  const unit = match[2].toLowerCase();
  return Math.round(value * INTERVAL_UNITS[unit]);
}
```

### Examples

```bash
# Every 30 seconds (health checks)
npx tsx src/cron-cli.ts add \
  --name "api-health" \
  --every "30s" \
  --instruction "Ping API endpoint and log response time"

# Every 5 minutes (frequent polling)
npx tsx src/cron-cli.ts add \
  --name "queue-monitor" \
  --every "5m" \
  --instruction "Check queue depth and alert if over threshold"

# Every 2 hours (periodic sync)
npx tsx src/cron-cli.ts add \
  --name "data-sync" \
  --every "2h" \
  --instruction "Sync customer data from CRM"

# Every day (using interval instead of cron)
npx tsx src/cron-cli.ts add \
  --name "daily-backup" \
  --every "1d" \
  --instruction "Backup database to S3"
```

<Note>
  Interval jobs compute next run based on `last_run_at`, ensuring consistent spacing even if execution takes time.
</Note>

## Database Schema

From `~/workspace/source/supabase/migrations/003_add_cron_jobs.sql:4-38`:

```sql
create table if not exists warden_cron_jobs (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  enabled boolean not null default true,

  -- Schedule type: 'cron' (recurring), 'at' (one-shot), 'every' (interval)
  schedule_type text not null check (schedule_type in ('cron', 'at', 'every')),
  cron_expression text,        -- e.g. '0 9 * * *' (for schedule_type = 'cron')
  cron_timezone text not null default 'UTC',  -- IANA timezone
  at_time timestamptz,         -- one-shot fire time (for schedule_type = 'at')
  every_ms bigint,             -- interval in milliseconds (for schedule_type = 'every')

  instruction text not null,   -- task instruction to run
  task_metadata jsonb,         -- flows to created task

  last_run_at timestamptz,
  next_run_at timestamptz,
  last_task_id uuid references warden_tasks(id) on delete set null,
  run_count int not null default 0,

  delete_after_run boolean not null default false,

  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

-- Index for efficient polling of due jobs
create index idx_warden_cron_jobs_next_run
  on warden_cron_jobs (next_run_at)
  where enabled = true;
```
